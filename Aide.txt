AI‑IDE Master Build Plan
(Architecture, Modules, Data Structures, and Development Phases)

1. Core Goals
Build a C++ AI‑powered IDE with:

Full text editor (undo/redo, diff, syntax, LSP).

Differential editor for AI patches.

File browser (local + remote).

AI orchestration (Ollama local + Gemini + Claude + Copilot).

Action logging + reversible AI changes.

Prompt rewriting + re‑run.

Git integration.

Time‑travel features (snapshots, history, persistent undo).

3. Core Data Structures
3.1 Patch
Code
struct PatchEdit {
    Range range;
    std::string oldText;
    std::string newText;
};

struct Patch {
    std::string filePath;
    std::vector<PatchEdit> edits;
};
3.2 AIAction
Code
struct AIAction {
    std::string id;
    std::string provider;
    std::string model;
    std::string prompt;
    std::string contextSummary;
    std::vector<Patch> patches;
    std::string timestamp;
    bool applied;
};
3.3 AIRequest / AIResponse
Code
struct AIRequest {
    std::string mode; // autocomplete, refactor, explain, etc.
    std::string prompt;
    std::string fileContext;
    std::string selection;
    bool streaming;
};

struct AIResponse {
    std::string text;
    std::optional<Patch> patch;
};
